Tomcat笔记

Tomcat

因为tomcat都是java写的，
所以批处理文件执行的时候
他会需要找到javahome变量
没有的话startup就会一闪而过

tomcat的报错信息
可以在他的logs目录下查看日志信息

  1. 一般将tomcat的端口号改为：80

    • 原因：80端口是http协议的默认端口号
      在访问的时候不需要输入localhost：后面的80
      直接浏览器输入localhost就可以了
    • 这就是我当时为什么刷新的时候端口号不见了的原因

  2. 关闭

    • 强制关闭就是点x
    • 正常关闭就是点那个shutup
    • 或者在startup里输入CTRL+c

  3. 访问

    • 比如webapps里面有一个hello项目，访问的时候直接

    • 输入localhost\hello 就可以–虚拟路径

    • 部署项目的时候把项目打成一个war包（压缩的形式）
      然后复制到webapps里面
      然后他就会自动的解压缩
      删除war包的时候也可以自动删除

    • 在conf\catalina\localhost创建任意名称的xml文件。在文件中编写

      * 虚拟目录：xml文件的名称

  1. 结构

    • 动态静态的文件项目
    1. 如果项目里面有WEB-IFO说明就是动态的项目
      – WEB-IFO
      --  里面有web.xml 核心配置文件
-- classes目录：放置字节码文件
-- lib目录：放置项目依赖的jar包
2. 利用开发工具部署

    -- 有热部署
        -- 在配置里改成update资源的时候更新
        -- 不然的话，需要重新启动服务器

servlet：sever applet

* 概念： 运行在服务器上的小程序
    * servlet 接口 定义了java类被浏览器访问到（tomcat识别）的规则
        -- A servelet is a small java program that runs within a Web server 
    * 将来我们自定义一个类，实现servelet接口，复写方法。 这就是称之为一个serverlet

* 快速入门
    * 创建一个javaee项目
        -- 然后写吧！ implements servelet
