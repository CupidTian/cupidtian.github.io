Linux学习笔记

2019年11月24日23:12:01

  • Linux是Linus·s Unix改过来的的

  • Windows

    • 单用户，多任务

  • Linux

    • 多用户，多任务

      没有单任务的计算机现在

  • 服务器的稳定性非常的好

    一台服务器可以1000多天不关机

  • linux是内核

    • 分支是Ubuntu之类的

命令学习

  • shutdown
    • shutdown -h now
  • halt
  • init 0

  • VM备份

    • 快照：又称还原点，就是在保存快照时的系统状态，在后期可以随时恢复

      • 【侧重点在于短期备份、需要频繁的使用备份】
      • 做快照的时候虚拟机操作系统一般在开启
      • 看演示非常简单

    • 克隆：【侧重于长期备份】

      • 一定在关闭状态上

    • 我的理解：

      • 大概就是快照在自己电脑上，克隆可以转到别人电脑上
      • 快照大概就是虚拟的文件，克隆是实体的文件，跟老师上次发的一样
  • 日常运维的操作，有一半的精力都是对文件进行操作
    • Linux本身就是一个基于对文件操作的系统
    • 目录就是文件夹
    • Linux一切皆文件
  • 目录结构
    • Bin：全称Binary，含义是二进制。该目录存储的都是一些二进制文件
    • Dev：该目录主要存放的是外接设备，比如插入一个U盘，被识别在Dev目录下
      外接设备不能直接被使用，需要挂载
      在windows里，插入U盘自动给 识别成U盘或H盘，但咱Linux下要手动，这个叫分配盘符（挂载）
    • Etc：主要是存储一些配置文件。
    • Home：除了root用户以外，其他用户的家目录，类似于windows下User/用户目录
    • proc：process：进程目录。不要打开这个目录，因为里面的东西太多了，会卡屏
    • root：是root用户自己的家目录
    • Sbin：Super binary，里面也是二进制文件，但是必须有super权限的用户才执行
    • tmp：临时目录
    • usr：存放用户自己安装的软件，windows，类似于windows下program files
    • var：存放系统日志
    • mnt：当外接设备需要挂载的时候，需要挂载到mnt目录下

  • linux指令

    • 一些命令

      * 基础指令
    * ls
    * pwd
    * cd 
    * mkdir
    * touch
    * cp
    * mv 
    * rm
    * vim
    * 输出重定向
    * cat

    • ls：（list）

      • 这些选项的顺序都是无所谓的

      • 列出所以文件和文件夹

      • 相对路径的写法：经常使用2个符号“./”【当前目录下】,”../“【上一级目录下】

        • ls root 和 ls ./root一样此时

        • ls -l 以详细列表的形式列出，列出的有权限信息，修改时间，字节大小等

          • “-、d” 开头 是表示文件的信息，是文件夹还是文件
            • “-”是文件
            • “d”是文件夹

        • ls -la a的表示是列出所有的文件、文件夹，包括隐藏文件

          • 以上这两个用的非常的多
          • 隐藏文件一般以“.”开头

        • ls -lah

          • h的含义就是以更有可读性的形式来输出文件、文件夹的信息（其实就是文件大小，自动变成M、kb的形式）

      • 列出的文件注意一下颜色

        • 蓝色的表示文件夹
        • 黑色的表示文件
        • 绿色的表示拥有所有权限

    • pwd

      • print working directory 打印当前工作目录

    • cd

      • change directory
      • “~”代表家目录，直接进入root了(你现在是root用户的情况下，你要是jackietian，你进入的就是home/jackietian)

    • mkdir

      • make directory（创建目录、创建不了文件）
      • mkdir 路径
      • 假设你现在root下没有目录
        • mkdir 路径 表示在当前文件夹存在的条件下再创建一个文件（夹）
        • mkdir -p 路径： 表示在文件夹下比如说root/a/b/c/d 可以依次自动的创建
        • mkdir 路径1 路径2 路径3 ： 可以创建多个文件，但是这个用的比较少

    • touch

      • touch 路径

    • cp

      • cp 被目标路径 目标路径
      • 记住：这样复制的时候还可以顺便改一下目标名字（但一般不这样做）
      • 如果要复制文件夹的时候 要加入-r参数 否则会略过
        • -r 表示递归复制

    • mv

      • 移动、重命名
      • move
        • mv 需要移动的文档路径 目标位置
      • 这个也可以在移动过程中改名字
      • 移动不需要递归
      • 当然你原地移动的意思不就相当于重命名嘛！

    • rm

      • 移除命令
      • remove
        • rm 路径
          • 在删除的时候会提示要不要删除，让你输入y or n
          • 但是在Ubuntu下没有这个提示，会直接删除
        • rm -f 路径
          • -f的意思是强制删除
        • 注意，在删除目录的时候会不让你删，提示这是一个目录
          • 所以，就要用force了
          • 而且删除的是文件夹，还需要进行递归调用
        • rm -rf 文件夹路径
          • 这样就可以完成删除啦
        • rm -rf linux*
          • 可以把该文件夹下开头为linux的文件全部删除（不知道文件夹删除不）
          • 这个“ * ”此时的含义，是通配符

  • 技巧*

    • 如果一些路径的首字母什么的路径名是唯一的，那可以按tab进行补全
    • vim
      • 超强文本编辑器，文本编辑器之神
      • 通过vim 打开文件或不存在的文件
        • 退出的话按住 shift + : 然后输入q
    • 输出重定向
      • ls -lha > ls.txt可以新建一个文件把ls目录添加在ls.txt里面

        • 可以覆盖

        • 就是追加

    • cat
      • cat 文件路径
        • 直接在终端中打开一个文件，而不是像vim一样其实是一个新窗口再退出
      • cat 待合并路径1 待合并路径2 … 待合并路径n > 目标文件
        • 可以合并多个文件到一个文件下，非常的神奇

Spring学习

Spring学习

2019年11月16日15:23:47

  1. Spring的优点

    • 高内聚低耦合

      • Spring相当于一个大工厂（容器），讲对象创建和依赖关系维护，交给Spring进行管理
      • Spring的作用就是用于生成Bean

    • AOP切面编程的支持

  1. 入门案例：Ioc

    • 导入Jar包

      • 4+1 四个核心(beans,core,context,expression)，一个依赖(commons.loggins )，这个beanutils必须要有这个依赖

        • .jar(字节码)
        • javadoc.jar(文档)
        • source.jar(源码)

    • 之前的开发中 要用new一个对象
      学习Spring后，将由Spring创建对象实例–>IoC(控制翻转，inverse of control)创建对象的权限由自己翻转给了spring
      之后需要实例对象的时候，从spring工厂获得，需要将实现类的全限定名称配置到xml文件中

* 先写一个接口

* 再写一个他的实现类

* 进行配置文件约束
    * 位置：任意，开发中一般在classpath下面（src）
    * 名称：任意，开发常用applicationContext.xml（约定俗成，你看就是你用的Spring）
    * 内容：添加schema约束：
    约束文件位置：spring-framework-4.3.2.RELEASE\docs\spring-framework-reference\html\xsd-configuration.html
    打开这个网页，里面就有xml源码
    

* 开始配置
    * 你实现的接口类以后要交给spring了
        
        * <bean>是配置需要创建的对象
            id：用于之后从spring容器获得实例时使用的
            class：需要创建实例的全限定类名

  1. 入门案例：DI

    • DI： dependency injection 依赖注入
      is a:是一个，继承，子类就是那个父类
      has a：有一个，有一个成员变量，依赖。

      • 依赖：一个对象需要另一个对象
      • 题外话：突然发现JUnit好像是单元测试的框架，回头去玩玩
      • 注入：通过setter方法进行对另一个对象实例的设置
